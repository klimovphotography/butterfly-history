# Telegram бот для сайта «Эффект Бабочки»

Этот бот повторяет основную механику сайта:

- принимает вопрос в формате `Что если ...?`
- поддерживает режимы: `Реализм`, `Мрачная хроника`, `Эпоха процветания`, `Безумие`, `Юмор`
- строит сценарий альтернативной истории
- показывает таймлайн по годам
- дает кнопки продолжения ветки
- отправляет 1-2 картинки, если доступен `GEMINI_API_KEY`

## Что уже готово

- основной файл бота: `bot.mjs`
- логика генерации сценариев: `lib/alt-history-core.mjs`
- хранение состояния чатов: `data/state.json` создастся автоматически

## Что вам нужно сделать

1. Зайдите в папку бота:

```bash
cd "/Users/vasilijklimov/Documents/Codex project/telegram bot"
```

2. Создайте `.env`:

```bash
cp .env.example .env
```

3. Вставьте токен вашего телеграм-бота:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_бота
```

4. Если AI ключи уже лежат в корневом `../.env` сайта, больше ничего можно не добавлять.

5. Запустите бота:

```bash
npm start
```

`npm install` здесь не обязателен, потому что у бота нет внешних зависимостей.

## Команды бота

- `/start` - приветствие
- `/mode` - выбор режима
- `/random` - случайный пример
- `/reset` - очистить текущую ветку
- `/help` - помощь

## Как загрузить на VPS

Самый простой вариант:

1. Перенесите на сервер весь проект или минимум папку `telegram bot` плюс `.env` сайта.
2. На сервере перейдите в папку бота.
3. Убедитесь, что установлен `Node.js 18+`.
4. Создайте `.env` с токеном.
5. Запустите `npm start`.

Если хотите, чтобы бот сам поднимался после перезагрузки сервера, используйте пример файла:

- `butterfly-history-bot.service.example`

## Автодеплой через GitHub

Если вы деплоите проект кнопкой/пушем в `main`, бот тоже поедет на сервер и перезапустится автоматически.

Для этого я добавил общий PM2-конфиг:

- `/root/butterfly/ecosystem.config.cjs`

Важное правило безопасности:

- `.env` сайта должен лежать только на сервере: `/root/butterfly/.env`
- `.env` бота должен лежать только на сервере: `/root/butterfly/telegram bot/.env`
- в GitHub эти файлы отправлять не нужно и нельзя

Что должно быть в `.env` бота на сервере:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_бота
```

Если AI-ключи уже есть в `/root/butterfly/.env`, бот их сам подхватит.

После этого схема будет такой:

1. Вы меняете код.
2. Отправляете в `main`.
3. GitHub Actions делает деплой.
4. На сервере автоматически перезапускаются и сайт, и бот.

## Важно

- Бот работает через `long polling` (лонг-поллинг). Это значит, что не нужно настраивать webhook.
- Если бот перезапустится, его состояние диалогов останется в `data/state.json`.
- Если нет `GEMINI_API_KEY`, бот все равно будет работать, но без картинок.
